Hadoop 相关配置

完全分布式安装

  1. 宿主机作为master运行namenode和jobtracker，两个虚拟机作为slave运行datanode（tasktracker），虚拟机直接用hadoop需要使用的用户名作用户名。

  2. 宿主机：centos6.5 虚拟机：ubuntu10.10（暂时配置一台另一台克隆）分别安装java。
    BTY:centos可安装直接rpm，然后更改配置文件（rpm不配置也可以用）
    vim /etc/profile

    ubuntu用tar包，hadoop安装ubuntu可用deb包。
    建议都使用tar源码。

  3. 为hadoop使用添加用户,所有机器要求使用相同的用户名。

  4. 配置/etc/hosts。
    配置ssh
    启动服务

    su h（不要用错用户）
    cd /home/h
    ssh-keygen -t rsa

完成后，在home跟目录下会产生隐藏文件夹.ssh 内有公钥私钥。
两个slave结点的公钥中必须要包含主结点的公钥
ssh 拷贝过去：
scp authorized_keys os001:/home/L/.ssh/master_keys(重命名防止覆盖自身的公钥)
scp authorized_keys os002:/home/L/.ssh/master_keys
再在两个节点都执行（可以直接ssh执行）
cat master_keys >> authorized_keys
chmod 644 authorized_keys
664的权限不行！
或者
cat ~/.ssh/id_rsa.pub | ssh root@datanode1 "cat >>~/.ssh/authorized_keys"

  1. su h
    wget http://mirrors.cnnic.cn/apache/hadoop/common/hadoop-1.2.1/hadoop-1.2.1.tar.gz
    安装hadoop，使用hadoop的用户解压，最好不要用rpm。 最好保存在共有的目录。（先在宿主（不一定是master）机器上装好再scp过去，保持目录、配置目录一致）
    配置文件在/hadoop/conf下（rpm安装默认在/etc/hadoop，现在还有之前配置的旧的文件，移动到了L/hadoopconfig）。

  2. 创建文件夹/home/h/hadoop name tmp dat
    配置hadoop-env.sh
    java目录 /usr/java/jdk1.7.0_51
    配置：core-site.xml、hdfs-site.xml、mapred-site.xml

    • core-site.xml
    • hdfs-site.xml

    • mapred-site.xml

      编辑masters slaves。内容分别为各节点系统名：master os001 os002

  3. 先将datanode的hosts配置下，然后按master新建目录。
  4. 将hadoop scp在datanode。

需要root密码
datanode的命令可在 /usr/java/jdk…/bin 使用./执行

  1. 启动hadoop(hadoop专用用户执行):
    bin/hadoop namenode -format
bin/start-all.sh
    启动前可能需要先删掉name文件夹。
    可能会出现找不到配置文件的情况，重启下就ok，source 下估计也可以，再格式化、start-all就完成了了。
  2. jps等脚本再hadoop x.x.x.安装目录下，sh等脚本在hadoop x.x.x/bin/下。
    hadoop脚本放在了/usr/sbin/文件夹下，需要给ssh文件执行权限 644 —> 755 chown h /var/log/hadoop/h。
  3. 可以通过http://jobtracker:50030 查看下。
  4. sshd开机启动 sudo chkconfig --level 5 sshd on
  5. dfs.replication这个参数是个client参数:hdfs的副本数.
    如果你只有3个datanode，但是你却指定副本数为4，是不会生效的，因为每个datanode上只能存放一个副本。
  6. 在 etc/profile里面设置JAVA_HOME可以设置为usr/java/default，这是一个symbol link，可以避免因JAVA环境变量变化而引起的错误。
    rpm安装完毕后在/etc/hadoop里面设置hadoop-site.xml等文件
    将本地文件拷贝至HDFS上，此时应使用 hadoop fs –lsr / 来查看hdfs的文件夹内容，以确定不会读错文件。
  7. 测试。
    单词计数测试
    mkdir file
    cd file/
    echo “Hello World” > file1.txt
    echo “Hello Hadoop” > file2.txt
    在HDFS上创建输入文件夹
    bin/hadoop fs -mkdir input
    上传到dfs（文件系统）
    bin/hadoop 的fs -put ../file/file*.txt input
    bin/hadoop 的fs -ls input
    bin/hadoop jar ~/hadoop-1.2.1/hadoop-examples-1.2.1.jar wordcount input output
    这块出问题可能需要关闭slaves的防火墙
    chkconfig —level 35 iptables off（永久关闭）
    查看运行结果：
    bin/hadoop fs -ls output
    bin/hadoop fs -cat output/part-r-00000
    Hadoop 1
    Hello 2
    World 1
  8. 关于data文件夹权限（位于slave），在datanode上运行jps没有datanode进程，通过日志文件发现data文件权限不同。
    http://os001:50060/(其他目录也可以进去) datanode日志文件中有这么一行： Incorrect permission for /home/h/hadoop/data, expected: rwxr-xr-x, while actual: rwxrwxr-x 改成755就ok了。
    bin/stop-all.sh ssh os001 sudo chmod 755 data
    jobtracker 50030 监控jobtracker
    namenode 50070 监控集群
  9. groupadd hadoop
    usermod -a -G hadoop h #千万不要忘了-a
    usermod -a -G hadoop L
    chmod 770 h
    chmod 770 h/*
    (后9位又被分为三部分，每部分三个位置“—- —- —-”。第一部分的三个位置表示属主（也就是文件的主人或者说是创建者），第二部分的三个位置表示属组（也就是那些组里面的成员对这个文件有什么样的权限），第三部分是其他所有人。那么这每三个位置都可以有r(可读)w(可写)x(可执行)的权限。
    )

Hadoop使用常见问题以及解决方法

hortonworks或者cloudera去搭建系统

实际上用hadoop的用户启用eclipse即可，eclipse的workspace设置在公开的区域，发现新用户需重新配置输入法等。

h的主文件夹权限也有要求
drwx———. 30 h h 4096 5月 9 20:03 h
若hadoop在h的文件夹内其它用户就不能通过eclipse使用hadoop的jar包。

此时更改权限（添加到一个用户组，为用户组开放部分权限）也不可以。
可将hadoop，hadoop目录都复制一份。

再更改用户组

chmod 755 hadoop-1.2.1
chmod 755 HadoopDir/
chown -R h hadoop-1.2.1
chown -R h HadoopDir/
chgrp -R hadoop hadoop-1.2.1/
chgrp -R hadoop HadoopDir/

这里ssh跟hadoop有矛盾，hadoop的安装目录要求在hadoop专用的用户主文件夹下（也可以不是？），好复制。但当前登录的用户访问不了这个目录，若是开放这个主文件夹和其子文件夹，ssh因为权限问题就不能免密码登录了（还有eclipse 的问题）。所以还是复制到当前用户，当前用户作为hadoop用户。
chown -R L HaoopDir
chown -R L hadoop-1.2.1
其它os的也改用户名
usermod -l L -d /home/L -m h
groupmod -n L h
passwd
passwd L
ssh等等都重新來。
刚开始就用登录用户。或者配置用户下的输入法也一样，甚至更好。

Hadoop集群在namenode格式化（bin/hadoop namenode -format）后重启集群会出现如下 Incompatible namespaceIDS in … :namenode namespaceID = … ,datanode namespaceID=… 错误， 原因是格式化namenode后会重新创建一个新的namespaceID,以至于和datanode上原有的不一致。 解决方法：
（1）删除datanode dfs.data.dir目录（默认为tmp/dfs/data）下的数据文件(在datanode上，具体位置查看hdfs.site文件 rm -rf /home/L/HadoopDir/data/ ) ，再重新格式化。 bin/hadoop namenode -formate
（2）修改/current/VERSION 文件，把namespaceID修成与namenode上相同即可（log错误里会有提示）
（3）重新指定新的目录
