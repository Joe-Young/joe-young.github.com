git commit -m “Add cute octocat story” Store the staged changes

git commit -m ‘Add all the octocat txt files’
git status

git diff HEAD the diff of our most recent commit, which we can refer to using the HEAD pointer.

Cassandra理解--存储

存储方式

Hbase与Cassandra存储方式的对比

Cassandra与Hbase的存储形式是不同的。Hbase是搭建在HDFS这种分布式文件系统上的，本身并不是太需要进行分布式的存储管理，因为其可视HDFS为本地文件系统，不考虑具体数据存放在哪些节点，其功能主要是在HDFS基础上进行优化（针对HDFS增加和修改等问题），存储方式相对简单。Cassandra不同，Cassandra不需要建立在HDFS这种分布式文件系统上，故其需要考虑具体数据分布的问题，自行设计存储的模型。

Cassandra的存储的模式是什么样的呢？

环状。Cassandra的每一个DataCenter是一个环形（以下不做特殊说明均指统一DC），不同于HBase的星形（中心型）—HDFS就是中心型。数据中心的所有节点共同在这个环上，每个节点首尾相连，组成一个环。由此看Cassandra是去中心化的，不存在master（存在seeds，稍后介绍），所以集群的启动方式是加入式，即节点是启动后加入到环中的（不同于HBase集中式一起启动）。可是每个节点在这个环的什么位置呢？这个由cassandra配置文件的token决定。如果不使用vnode，这个token值需要自己设置。token的取值在-2^63~2^63之间。
